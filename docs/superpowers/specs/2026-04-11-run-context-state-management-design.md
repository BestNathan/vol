# RunContext 状态管理重构设计

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 Agent 内部状态管理迁移到 RunContext，使 `run()` 方法直接从 RunContext 构造并返回 `AgentResponse`。

**Architecture:** 
- 在 `RunContext` 上新增状态收集字段（reasoning_chain、tool_call_records、final_content、error）
- 新增手动记录方法，agent loop 显式调用这些方法记录状态
- 新增 `finalize()` 方法，从 RunContext 构造 `AgentResponse`
- `AgentResponse.reasoning` 从 `String` 改为 `Vec<ReasoningStep>`

**Tech Stack:** Tokio async runtime, Arc<RwLock> for thread-safe state mutation

## 1. 新增类型 ReasoningStep

### 文件：`crates/vol-llm-agent/src/react/state.rs` (新建)

```rust
//! Agent internal state types.

use chrono::{DateTime, Utc};
use serde::{Deserialize, Serialize};

/// A single reasoning step in the agent's thought chain
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct ReasoningStep {
    /// Iteration number (1-based)
    pub iteration: u32,
    
    /// The thinking content generated by LLM
    pub thinking: String,
    
    /// Timestamp when this step was recorded
    pub timestamp: DateTime<Utc>,
    
    /// Duration of LLM call in milliseconds (if available)
    pub duration_ms: Option<u64>,
}

impl ReasoningStep {
    pub fn new(iteration: u32, thinking: String, duration_ms: Option<u64>) -> Self {
        Self {
            iteration,
            thinking,
            timestamp: Utc::now(),
            duration_ms,
        }
    }
}
```

### 文件：`crates/vol-llm-agent/src/react/run_context.rs`

#### 新增字段

```rust
use super::state::{ReasoningStep, ToolCallRecord};

pub struct RunContext {
    // 现有字段保持不变...
    
    // 新增：内部状态收集
    pub(crate) reasoning_chain: Arc<RwLock<Vec<ReasoningStep>>>,
    pub(crate) tool_call_records: Arc<RwLock<Vec<ToolCallRecord>>>,
    pub(crate) final_content: Arc<RwLock<Option<String>>>,
    pub(crate) error: Arc<RwLock<Option<String>>>,
}
```

#### 新增方法

```rust
impl RunContext {
    /// Record a reasoning step
    pub async fn record_reasoning_step(&self, thinking: String, duration_ms: Option<u64>) {
        let iteration = self.iteration.load(std::sync::atomic::Ordering::SeqCst);
        let step = ReasoningStep::new(iteration, thinking, duration_ms);
        self.reasoning_chain.write().await.push(step);
    }
    
    /// Record a tool call
    pub async fn record_tool_call(&self, record: ToolCallRecord) {
        self.tool_call_records.write().await.push(record);
    }
    
    /// Set final answer content
    pub async fn set_final_content(&self, content: String) {
        *self.final_content.write().await = Some(content);
    }
    
    /// Set error information
    pub async fn set_error(&self, error: String) {
        *self.error.write().await = Some(error);
    }
    
    /// Build AgentResponse from collected state
    pub fn finalize(&self) -> AgentResponse {
        use futures::executor::block_on;
        
        let reasoning_chain = block_on(self.reasoning_chain.read());
        let tool_call_records = block_on(self.tool_call_records.read());
        let final_content = block_on(self.final_content.read());
        let error = block_on(self.error.read());
        
        AgentResponse {
            content: final_content.clone().unwrap_or_default(),
            reasoning: reasoning_chain.clone(),
            run_id: self.run_id.clone(),
            session_id: self.session_id.clone(),
            iterations: self.iteration.load(std::sync::atomic::Ordering::SeqCst),
            tool_calls: tool_call_records.clone(),
            error: error.clone(),
        }
    }
    
    /// Get a clone of the reasoning chain
    pub async fn get_reasoning_chain(&self) -> Vec<ReasoningStep> {
        self.reasoning_chain.read().await.clone()
    }
    
    /// Get a clone of tool call records
    pub async fn get_tool_call_records(&self) -> Vec<ToolCallRecord> {
        self.tool_call_records.read().await.clone()
    }
}
```

#### 更新 `new()` 方法

```rust
pub fn new(
    run_id: String,
    user_input: String,
    session_id: String,
    session: Arc<Session>,
    tools: Arc<ToolRegistry>,
    config: AgentConfig,
) -> (Self, mpsc::Receiver<PluginRequest>) {
    let (event_tx, _) = broadcast::channel(100);
    let (plugin_event_tx, plugin_event_rx) = mpsc::channel(100);

    let ctx = Self {
        run_id,
        user_input,
        session_id,
        iteration: AtomicU32::new(0),
        messages: Arc::new(RwLock::new(Vec::new())),
        all_tool_calls: Arc::new(RwLock::new(Vec::new())),
        current_tool_calls: Arc::new(RwLock::new(Vec::new())),
        data: Arc::new(RwLock::new(HashMap::new())),
        session,
        tools,
        config,
        event_tx,
        plugin_event_tx,
        // 新增字段初始化
        reasoning_chain: Arc::new(RwLock::new(Vec::new())),
        tool_call_records: Arc::new(RwLock::new(Vec::new())),
        final_content: Arc::new(RwLock::new(None)),
        error: Arc::new(RwLock::new(None)),
    };
```

## 5. agent.rs 变更

### 文件：`crates/vol-llm-agent/src/react/agent.rs`

#### 移除局部变量

删除 agent_task 内的局部收集器：
```rust
// 删除这些行

// let mut tool_call_records = Vec::new();
```

#### 使用 RunContext 方法记录状态

在 LLM stream 消费后：
```rust
// Collect reasoning chain
if !thinking.is_empty() {
    reasoning_chain.push(format!("[Iteration {}] {}", iteration, thinking));
}
```

改为：
```rust
// Record reasoning step
if !thinking.is_empty() {
    run_ctx.record_reasoning_step(thinking, None).await;
}
```

在工具执行后：
```rust
// Record successful tool call
tool_call_records.push(crate::react::response::ToolCallRecord {
    tool_name: call.name.clone(),
    arguments: call.arguments.clone(),
    result: result.content.clone(),
    iteration,
    success: true,
});
```

改为：
```rust
// Record tool call
run_ctx.record_tool_call(ToolCallRecord {
    tool_name: call.name.clone(),
    arguments: call.arguments.clone(),
    result: result.content.clone(),
    iteration,
    success: true,
}).await;
```

在设置最终回答时：
```rust
// Store final response data
let final_content = content.clone();
let final_iterations = iteration;
```

改为：
```rust
// Store final response data
run_ctx.set_final_content(content.clone()).await;
```

在错误处理时：
```rust
return Err(crate::AgentError::ToolExecution {
    tool: call.name.clone(),
    error: e.to_string(),
});
```

改为：
```rust
run_ctx.set_error(format!("Tool execution failed: {}", e)).await;
return Err(crate::AgentError::ToolExecution {
    tool: call.name.clone(),
    error: e.to_string(),
});
```

## 边界情况处理

### 1. finalize() 中的 async 问题

`finalize()` 是同步方法，但需要读取 async RwLock。使用 `futures::executor::block_on` 是可行的，因为此时 agent loop 已完成，不存在阻塞 event loop 的风险。

### 2. 错误信息同步

当 `run()` 返回 `Err(AgentError)` 时，error 字段也会通过 `set_error()` 记录到 RunContext。调用者可以从 `AgentResponse.error` 或 `Result::Err` 获取错误信息。

### 3. 向后兼容

`AgentResponse.reasoning` 从 `String` 改为 `Vec<ReasoningStep>`，需要更新所有使用该字段的代码：
- `service.rs` 中的 `response.content` 不受影响
- 测试中的响应构造需要更新
- 示例代码需要更新
